# Sidebar Navigation Doctrine

Phase: 6.8J.0

## Objective

The DAL sidebar should reflect the constitutional workflow and the business workflow, not the order in which development tools were created.

## Core Rule

Navigation clarity is not authority. Sidebar grouping does not create persistence, lifecycle transitions, ScopeVersions, server routes, or execution authority.

## Workflow Priority

Business users should see the customer and opportunity path before graph-oriented development tools:

Customer
↓
Opportunity
↓
Translate
↓
Scope Review
↓
Prism
↓
Quote

## Functional Preservation

No workspace is deleted. Existing development workspaces remain reachable, with graph and legacy analysis tools grouped under Advanced.

## Boundary

This phase reorganizes navigation configuration only. Placeholder entries use the existing DAL placeholder component until their composed workspaces are implemented.
