# Corridor Lens Types

Status: lens type doctrine.

## Lens Types

| Lens | Purpose |
| --- | --- |
| `HYPERSCALER` | Evaluate AI-scale demand, power, interconnection, expansion land, and scalable infrastructure. |
| `NEOCLOUD` | Evaluate GPU-ready deployment, power, interconnection, and transport fit. |
| `ENTERPRISE` | Evaluate enterprise buildings, parks, public institutions, serviceability, and commercial return. |
| `DUCT_MONETIZATION` | Evaluate duct, innerduct, ROW, wireless, municipal, utility, and adjacent-customer monetization. |
| `DARK_FIBER_IRU` | Evaluate fiber availability, pair assignability, data center/carrier demand, and IRU potential. |
| `TRANSPORT` | Evaluate lit transport, POPs, backbone nodes, cloud/carrier handoff, regen, and ADM fit. |
| `INTERCONNECTION` | Evaluate carrier hotels, IX, cloud on-ramps, meet-me rooms, and handoff capability. |
| `POWER_AI_EXPANSION` | Evaluate power, substations, transmission, generation, land, and AI expansion optionality. |
| `MUNICIPAL` | Evaluate public-sector sites, schools, ROW, utilities, and community anchor connectivity. |
| `UTILITY` | Evaluate utility infrastructure, easements, power adjacency, and utility customer opportunities. |
| `CARRIER_WHOLESALE` | Evaluate carrier wholesale demand, carrier hotels, POPs, dark fiber, duct, and transport. |

## Per-Lens Requirements

Each lens defines:

- purpose.
- primary objects.
- secondary objects.
- ignored or low-priority objects.
- preferred providers.
- scoring emphasis.
- monetization relevance.
- risks to elevate.
