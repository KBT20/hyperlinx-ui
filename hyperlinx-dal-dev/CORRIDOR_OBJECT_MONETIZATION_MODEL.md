# Corridor Object Monetization Model

Status: monetization doctrine only.

## Monetization Principles

Objects can create direct revenue, residual value, expansion options, or cost avoidance.

Monetization evidence does not create commercial authority. Marketplace, commercial review, and Route Engineering remain future downstream authorities.

## Object Examples

| Object | Potential monetization impact |
| --- | --- |
| `CONDUIT` | Duct sale |
| `INNERDUCT` | Duct sale and residual capacity |
| `FIBER` | Dark fiber IRU and transport revenue |
| `FIBER_PAIR` | Assignable dark fiber or transport inventory |
| `DATA_CENTER` | Transport revenue and interconnection demand |
| `SUBSTATION` | AI expansion potential |
| `TRANSMISSION_LINE` | AI fabric and power-proximity value |
| `PARCEL` | Future campus opportunity |
| `DEVELOPMENT_SITE` | Future campus or data center expansion |
| `CARRIER_HOTEL` | Interconnection revenue |
| `IX` | Peering and interconnection relevance |
| `CLOUD_ONRAMP` | Hyperscaler and cloud transport opportunity |
| `LSO` | Metro aggregation and transport revenue |
| `WIRELESS_SITE` | Backhaul and aggregation opportunity |
| `DUCT_OPPORTUNITY` | Direct duct sale |
| `DARK_FIBER_OPPORTUNITY` | Dark fiber IRU |
| `TRANSPORT_OPPORTUNITY` | Recurring transport revenue |
| `IRU_OPPORTUNITY` | Long-term fiber or duct IRU |
| `EXPANSION_OPPORTUNITY` | Future capacity and expansion revenue |

## Monetization Categories

The catalog uses:

- `DUCT_SALE`
- `DARK_FIBER_IRU`
- `TRANSPORT_REVENUE`
- `INTERCONNECTION_REVENUE`
- `AI_EXPANSION_POTENTIAL`
- `FUTURE_CAMPUS_OPPORTUNITY`
- `RESIDUAL_CAPACITY`
- `COST_AVOIDANCE`
- `NONE`

## Boundary

Objects identify monetization potential. They do not create quotes, contracts, revenue truth, or ScopeVersions.
