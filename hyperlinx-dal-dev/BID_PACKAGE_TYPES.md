# Bid Package Types

Status: doctrine and contracts only.

## Supported Types

| Type | Purpose |
| --- | --- |
| `FULL_PROJECT` | Entire approved project or corridor scope |
| `SEGMENT` | Route segment or milepost range |
| `STATION_GROUP` | A group of stations |
| `DISCIPLINE` | Work grouped by discipline |
| `CATEGORY` | Work grouped by object or commercial category |
| `HYBRID` | Combined structure across categories, segments, or disciplines |

## Examples

### Full Project

Example:

Dallas to Kansas City project.

Includes:

- conduit.
- fiber.
- splicing.
- optical deployment.
- crossings.

### Segment

Example:

MP0-MP50 segment.

Includes measurable work within a specific route segment.

### Station Group

Example:

Stations 100-150.

Includes items tied to specific station references.

### Discipline

Examples:

- Fiber Placement.
- Splicing.
- Optical Deployment.
- Permitting.
- Electrical Construction.

### Category

Examples:

- Conduit package.
- Power package.
- Interconnection package.
- AI corridor package.

### Hybrid

Example:

Conduit + Fiber package.

Hybrid packages are useful when related work should be priced together but still retain station and segment allocation.
