# Google Helium RFP Doctrine

The Google Helium RFP is modeled as a hyperscaler bid-response opportunity. It creates requested route requirements, response artifacts, budgetary estimates, and staging outputs for human review.

Source artifact structure observed:

- Google Helium workbook tabs: Cover Page, A Requirements, B Location List, C Network Spans for Bid, D Vendor Response.
- RFP package KMZ artifacts: HIU-Summary-06-03-2026.kmz, MUS 07162024.kmz.
- Prior cost precedent: Google Fiber Project - 20251121.xlsx, Dobson ILA Cost Summary 27 vs 36 Racks.xlsx.

Google-specific requirements:

- Helium / HIU near Dodge City, KS to Muskogee / MUS.
- Helium / HIU near Dodge City, KS to Stillwater / SWR.
- Stillwater route must be compared against Muskogee route for geographic diversity readiness.

No workbook write, email, submission, ScopeVersion creation, or inventory mutation occurs in this phase.
