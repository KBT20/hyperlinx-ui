# Prism Route Engineering Handoff Draft

The Route Engineering Handoff Draft packages advisory Prism output for human review.

It does not create a ScopeVersion.

It does not approve.

It does not certify a route.

It does not create Control work.

## Fields

- candidateId
- corridorId
- lensType
- networkRole
- referenceArchitectureId
- recommendationLevel
- requiredObjects
- requiredTools
- requiredDesignStandards
- humanReviewBlockers
- requiredRouteEngineeringReviews
- evidenceIds
- notes
- status

## Status

`DRAFT_ONLY`

## Doctrine

The draft tells Route Engineering what must be reviewed.

Route Engineering decides whether to accept, redline, request evidence, create an exception, reject, or prepare a future executable handoff.
