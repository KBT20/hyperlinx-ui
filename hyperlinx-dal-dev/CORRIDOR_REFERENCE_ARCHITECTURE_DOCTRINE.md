# Corridor Reference Architecture Doctrine

A customer ask should map to a reference architecture before it maps to execution.

This phase defines reusable reference architectures only. It does not implement routing, scoring, recommendations, provider calls, persistence, UI, ScopeVersion creation, or kernel changes.

## Core Doctrine

A lens determines what matters.

A reference architecture defines what should be considered.

Design standards define how objects must be treated.

Route Engineering reviews, redlines, approves, or rejects.

Sales may express intent.

Sales may not define final architecture.

Prism may recommend in a future phase.

Reference Architecture guides design.

Route Engineering remains authority.

## Prism Recommendation Relationship

Reference Architecture Fit informs Prism Recommendation.

Prism Recommendation may use architecture fit to identify required objects, required tools, missing evidence, and required design standards.

Prism Recommendation remains advisory.

Human review is required.

Route Engineering remains authority.

## Purpose

Reference Architecture translates customer intent into a structured review frame:

- applicable lens
- applicable network role
- applicable corridor class
- required objects
- optional objects
- required design standards
- required tools
- topology expectations
- capacity model
- protection model
- expansion model
- commercial product context
- Route Engineering review boundary

## Non-Authority Rule

A Reference Architecture does not approve.

It does not promote.

It does not create ScopeVersions.

It does not replace Route Engineering.

It tells Route Engineering what must be considered based on the ask.

## Human Review Boundary

Route Engineering may:

- accept
- redline
- add tools
- remove tools
- request evidence
- create a standards exception
- reject architecture fit

## Constitutional Rule

Lens determines what matters.

Reference Architecture determines what must be considered.

Design Standards determine how objects must be treated.

Prism scores and recommends only after lens, architecture, object, and design-standard context exists.

Route Engineering remains authority.
