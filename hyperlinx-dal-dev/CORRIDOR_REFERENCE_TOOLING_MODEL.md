# Corridor Reference Tooling Model

Tool requirements identify which tools must be considered for a reference architecture.

They are requirements, not live integrations.

No provider calls occur in this phase.

- GraphHopper

- Google Roads
- Teralinx Internal Corridor Model

## Rule

Tools create evidence or review context.

Tools do not create authority.

Route Engineering remains authority.
