# Adapter Remediation Doctrine

Phase: 6.7C

Shadow Runtime identifies gaps. The Remediation Layer resolves those gaps into adapter recommendations.

## Doctrine

The Constitutional Runtime remains authoritative.

DAL remains unchanged.

Adapters bridge differences.

Remediation normalizes, reconciles, and plans. It does not create truth.

## Boundary Rules

- No DAL mutation.
- No persistence writes.
- No execution changes.
- No lifecycle mutation.
- No authority mutation.
- No close creation.
- No marketplace mutation.

## Remediation Output

The remediation layer produces:

- Gap classification
- Read-only normalization recommendations
- Reconciliation results
- Remediation plans
- Diagnostics

No automated fixes are performed.
